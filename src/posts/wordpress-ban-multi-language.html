如何让 WordPress 前台不加载多语言包
2012-08-28 ｜ # WordPress
▼ 文章目录

据周良所知 WordPress 是通过 getText 类库来实现多语言功能，但是这个库运行效率比较低，是 WordPress 运行速度的一个大瓶颈。由于我们平时用的主题可以通过修改主题文件直接汉化成中文，并且国内的 WordPress 博客使用的都是中文版本的主题，因此在 WordPress 网站前台加载语言包并不是必须的。我们可以通过禁用 WordPress 加载多语言包，来达到 WordPress 提速的目的。

方法很简单，通过修改 wp-config.php 文件来阻止 WordPress 在前台加载语言包。

打开 wp-config.php 文件，把 require_once(ABSPATH . 'wp-settings.php'); 改成下面的代码：

if(WP_ADMIN === true) {
define ('WPLANG', 'zh_CN');
} else {
define ('WPLANG', 'xxxxxxxxx');
}
require_once(ABSPATH . 'wp-settings.php');

这个方法原理是通过改变 WPLANG 常量使 WordPress 不能正确加载语言包从而提高运行速度。通过测试，去除语言包后，WordPress 运行速度可以提高 0.3～0.5s 左右。
