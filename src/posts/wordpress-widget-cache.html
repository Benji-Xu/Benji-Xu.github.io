为 WordPress 的 Widget 建立缓存
2012-12-31 ｜ # WordPress
▼ 文章目录
  1. 1. WP  widget  cache插件功能特点
  2. 2. WP widget cache插件安装：

WordPress 的 Widget（小工具）非常的强大，一般的 WordPress 博客， Widget 侧边栏都会用到最新文章、最新评论、友情链接等。大多数站点都会对文章进行缓存，但是侧边栏那？可能很少有人注意到（用静态页面缓存方法除外）。为了让你的 WordPress 站点飞起来，对 Widget 建立缓存也是必不可少的。

众所周知 WP-Cache、WP Super Cache 和 W3 Total Cache 这几款插件都是可以加速网页，也就是对文章之类的页面进行缓存。今天介绍插件和非插件的方法来缓存 WordPress 的侧边栏(SideBar) Widget 小工具，同样起到加速页面显示的作用，将会帮助我们提高侧栏 widget 的加载速度，让你的 WordPress 站点提速，飞起来！

WP  widget  cache插件功能特点

  • 支持设定时间更新缓存数据
  • 支持定义促发事件（例如：发布一篇文章）自动更新缓存数据
  • 支持对用户指定的边栏widget进行缓存处理（例如：你可以只缓存最新文章 widget，但不缓存随机文章 widget）
  • 安装设置简单，无须添加和修改源代码（比 WP-Cache 安装简单很多）

WP widget cache插件安装：

  • 下载 WP widget cache 插件，然后解压上传插件到 /wp-content/plugins/ 下的文件夹（通过后台安装也可）。
  • 到 WordPress 管理后台插件页面激活插件

当然，如果你不想使用插件的话，你也可以参考如下代码直接利用 Widget Class 里的 update_callback() 方法进行数据缓存。

class Test_Widget extends WP_Widget {
// 注册 Widget
function register() {
register_widget(‘Test_Widget’);
}

// 构造函数
function construct() {
$widget_ops = array(‘classname’ => ‘test_widget’, ‘description’ => ( ‘测试挂件’) );
parent::construct(‘test_widget’, (‘Test Widget’), $widget_ops);
}

// Widget 主体输出
function widget( $args, $instance ) {
extract( $args );
// 如果缓存已经过期，则重新生成结果
if(false === get_transient($args[‘widget_id’])) {
/

set_transient($args[‘widget_id’], $data, 3600);
} else {
echo get_transient($args[‘widget_id’]);

//……
}

// 保存 Widget 参数
function update( $new_instance, $old_instance ) {
$instance = $old_instance;
// 更新参数时删除已有的缓存
delete_transient($this->id);
//……
return $instance;
}

// Widget 设置面板
function form( $instance ) {
//……
}

// 下面这个函数是新增加的，用来删除缓存
function update_callback($widget_args = 1) {
// 删除缓存
delete_transient($this->id);
delete_transient($this->id);
// 调用原来的 update_callback() 方法，防止出错
parent::update_callback($widget_args);
}
}
