解决WordPress <?php if ( is_home() ) { ?> 无效的方法
2013-02-08 ｜ # WordPress
▼ 文章目录

最近在给博客修改模版。所以需要用到一些 WordPress 函数。在折腾到侧边栏特定页面显示时，遇到了一个的问题。所以在博客记录一下，避免以后犯同样的错误。

一般情况下，在PHP中，如果需要限制一段代码，只在首页显示。都会这样写：

<?php if ( is_home() ) { ?>
只会在首页显示的内容
<?php } ?>

is_home(); 函数在首页的时候会返回一个 true，这是常用的只在首页显示判断语句，可以插入任意一个 WordPress 主题页面中，常用于侧边栏 sidebar.php 或页脚 footer.php 等页面。

但是今天写主题时候突然发现用 <?php if ( is_home() ) { ?> 对侧边栏进行处理，居然没效果。当时蛮笨的，没往代码方面考虑，一直以为是我的 if 函数放错位置了，纠结了半天…实际上是因为：

它之前有个 query_posts(); 会让它失效
原因是 is_home  is_sidebar  is_aipsme 这种 is_ 前缀的都是基于主循环来判断,
而 query_posts(); 会让其偏离主循环

解决方案是在 is_home(); 之前加一个 wp_reset_query();

<?php wp_reset_query(); if ( is_home() ) { ?>
我这次真的是只会在首页显示
<?php } ?>
